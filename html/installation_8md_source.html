1 @section Installation Installation
2 ## Header Only Library
3 astrotypes is a header only library and so installation is a case of
4 using `#include` directives and setting the -I option on
5 your compiler to point to the astrotypes headers install location.
6 Note that the [dependencies](@ref Dependencies) will also need to be installed.
7 
8 ### Example: including astrotypes using git subtrees
9 If you are using git to mamange your repository one of the many strategies
10 available is called subtrees. Although we provide a subtree example we do not
11 necessarily condone its use. Each stategy has its own merits and
12 compromises and the user should satisfy themselves as to the most appropriate
13 one for their project.
14 
15 From the top level of your project, create a directory for putting in thirdparty
16 dependencies.
17 @code
18 mkdir thirdparty
19 @endcode
20 Now use git subtrees to install astrotypes
21 @code
22 git subtree add --prefix thirdparty/astrotypes https://github.com/PulsarSearchSoft/astrotypes.git master --squash
23 @endcode
24 
25 Any time you need to update the version of astrotypes you can use:
26 @code
27 git subtree pull --prefix thirdparty/astrotypes https://github.com/PulsarSearchSoft/astrotypes.git master --squash
28 @endcode
29 
30 The only thing left to do is to update your build system so that your thirdparty directory, and the required dependencies
31 are added to the include path.

33 ## Building the unit tests and examples: cmake
34 Cmake is a widely used, platform and toolchain agnostic build tool.
35 One of the features of cmake is that it provides out of source builds
36 so you don't mix up build products (libraries, binaries, etc) from the source code.
37 This is very powerful in other ways, you may want to build the same source using
38 different compilers and compare the results.
39 
40 Configuring your build with cmake:
41  - create a directory for your build products
42  @code
43  mkdir build
44  @endcode
45  - point cmake to the top of the astrotypes repository (along with any flags)
46  and it will generate the appropriate build files for your host system
47  (gnu make files on linux, nmake or .proj files for windows, etc)
48 
49  e.g to build a debug debug
50  @code
51  cmake -DCMAKE_BUILD_TYPE=debug <path_to_astrotypes_repository>
52  @endcode
53  e.g to build a fully optimised version
54  @code
55  cmake -DCMAKE_BUILD_TYPE=release <path_to_astrotypes_repository>
56  @endcode
57 
58  Note that cmake will do its best to determine where dependencies are
59  on your system. You can override this behaviour with special flags.
60  See the dependencies section for more details and options.
61 
62  - run make
63  what you actually run will depend on your platform, but on linux this
64  will be, by default, make.
65 
66  @code
67  make
68  @endcode
69 
70  If you want to see whats going on under the hood you can run
71 
72  @code
73  make VERBOSE=1
74  @endcode

76 @section Dependencies Dependencies
77 ### Boost
78 Boost Website: https://www.boost.org
79 
80 This project requires the boost_units library.
81 You can use your local, or system installation of boost libs for this, however for convenience we also provide a
82 packaged version of boost_units (and its dependencies) in the thirdparty directory of this repository.
83 
84 By default the cmake for building the tests and examples will look for boost in the following order:
85  - User defined variables:
86  @code
87  -DBoost_ROOT=<path_to_boost_root>
88  @endcode
89  or
90  @code
91  -DBoost_INCLUDE_DIRS=<path_to_boost_includes> & Boost_LIBRARY_DIRS=<path_to_boost_libs>
92  @endcode
93  - The usual system paths (/usr/local/lib, /usr/lib etc.)
94  - Use the the thirdparty version
95 
96 If you wish to force the use of the thridparty packaged version add the flag -DUSE_THIRDPARTY_BOOST=true to ignore any boost versions installed on the system.
97 Note that boost units is a header-only library and does not need to be built.
98 
99 Cmake Variable | Description
100 -----------------------|-------------
101 USE_THIRDPARTY_BOOST | Force the use of the packaged boost version, ignoring any installed boost versions (default: false)
102 Boost_LIBRARY_DIRS | Explicitly specify a directory where the boost libraries reside
103 Boost_INCLUDE_DIRS | Explicitly specify a directory where the boost header files reside
104 Boost_ROOT | Explicitly specify a directory where the boost distiburion resides
