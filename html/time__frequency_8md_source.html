3 ## Quick Start Example
4 An example where we have 8 bit data stored in spectrum order.
5 Each specturm has a 100 channels and there are 10000 spectrum in each block.
6 
7 ~~~~{.cpp}
8 #include "pss/astrotypes/types/TimeFrequency.h"
9 #include <algorithm>
10 
11 // construct a suitable object
12 TimeFrequency<uint8_t> time_frequency(DimensionSize<Frequency>(100), DimensionSize<Time>(10000));
13 
14 // fill it with 0 (by default the data is uninitialised)
15 std::fill(time_frequency.begin(), time_frequency.end(), 0);
16 
17 // Iterate over each spectrum using the [] bracket style interface
18 unsigned total_pow=0;
19 for(DimensionIndex<Time> i(0); i < time_frequency.dimension<Time>(); ++i) {
20  auto spectrum = time_frequency[i];
21  for(DimensionIndex<Frequency> j(0); j < spectrum.dimension<Frequency>(); ++j) {
22  total_pow += spectrum[j];
23  }
24 }
25 
26 ~~~~

33 ## Loading in FilterBank files

35 e.g.

37 #include "pss/astrotypes/types/SigProc.h"
38 sigproc::FileReader filterbank_file("my_filterbank_file.fil");
39 TimeFrequency<uint8_t> time_frequency;
40 
41 // The ResizeAdapter resizes the time_frequency object according to the filterbank files parameters and file size
42 filterbank >> ResizeAdapter<units::Time, units::Frequency>() >> time_frequency;
43 ~~~~

46 ## Would you care for a slice?
47 You can extract parts of a TimeFrequency chunk as a slice. The Slice provides a view of the underlying
48 data and does not make any copies.
49 
50 ~~~~{.cpp}
51 #include "pss/astrotypes/types/TimeFrequency.h"
52 #include <algorithm>
53 
54 // construct a suitable object
55 TimeFrequency<uint16_t> time_frequency(DimensionSize<Time>(10000), DimensionSize<Frequency>(100));
56 
57 // take a slice of data for all frequencies from the 10th to the 20th spectra
58 auto slice = time_frequency.slice(DimesionSpan<Time>(DimensionIndex<Time>(10), DimensionIndex<Time>(20));
59 
60 // slices have their own iterator interface
61 // e.g. fill the slice with zeros
62 std::fill(slice.begin(), slice.end(), 0);
63 
64 // you can also specify a slice of data using an index and a width, instead of two indicies
65 auto slice_2 = time_frequency.slice(DimesionSpan<Time>(DimensionIndex<Time>(0), DimensionSize<Time>(10));
66 
67 // you can further slice of spectra in Frequency to select a subset of the channels
68 auto reduced_channels_slice = slice_2.slice(DimensionIndex<Frequency>(10), DimensionSize<Frequency>(2));
69 
70 ~~~~

72 ## TimeFrequency or FrequencyTime?
73 The difference between the two classes is in the way the data is stored. The patterns of memory access
74 are often a major factor in the speed of any algorithm you may write and so your choice of class will also
75 be algorithm dependendent for maximal efficiency.
76 
77 But you don't have to commit to a specific type. By design the two classes have identical interfaces and
78 so any code you write for one should work for the other seamlessly (if perhaps not at optimal efficiency).
79 
80 You can convert between the two types using the tranpose constructor. e.g.
81 ~~~~{.cpp}
82 // example conversion from TimeFrequency ordering to FrequencyTime ordering of 8bit data
83 TimeFrequency<uint8_t> time_frequency(DimensionSize<Frequency>(100), DimensionSize<Time>(10000));
84 FrequencyTime<uint8_t> frequency_time(time_frequency); // all the data will be copied from the time_frequency object
85 ~~~~
86 
87 A simple way to write a function that can work with either type is to use a template.
88 
89 ~~~~{.cpp}
90 #include "pss/astrotypes/types/TimeFrequency.h"
91 
92 template<class TimeFrequencyType>
93 void fill_with_one(TimeFrequencyType const& data)
94 {
95  // use any of the common TimeFrequency/FrequencyTime interface methods here
96  std::fill(data.begin(), data.end(), 1U);
97 }
